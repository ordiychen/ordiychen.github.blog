计算器数据拷贝过程

零拷贝(Zero Copy):描述了计算机操作：CPU不执行将数据从一个存储区复制到另一个存储区的任务。通过网络传输文件时，通常用于节省CPU周期和内存带宽。

传统方式与零拷贝方式

传统方式实现

以Socket client发送文件为例，这个过程需要从一个文件中读取数据并将数据传输到SocketServer上（这个场景表述了很多服务器应用程序的行为，包括FTP服务器，CDN服务器，邮件服务器等）。

图1 数据的拷贝过程

这个拷贝过程的每一个文件片段(这里是：4096 byte)需要经历4个次用户模式和内核模式的上下文切换，而且操作完成前数据倍复制了4次。时序图如下所示：

图2. 用户上下文和 Kernel 上下文切换：

使用Kernel的高速内核缓冲区可以提升性能。但是如果数据量大于内核缓冲区时（例如视频文件、音频文件、卫星遥感图像等)，这个拷贝过程可能成为一个性能瓶颈(性能损耗的原因：CPU上下文切换的损耗，数据拷贝的IO wait)。

数据传输-ZeroCopy方法

在传统方法中，第2次 CPU Copy和第3次 CPU Copy是多余的，整个传输过程可以交给DMA(直接存储器,Direct Memory Access)完成。

图3. ZeroCopy数据拷贝过程实现

ZeroCopy拷贝过程时序图：

该实现依赖于OS内核的具体实现，以Linux 2.4+版本为例，kernal对实现了sendfile + DMA gathger可以实现将文件描述符之间的拷贝。整个过程发生了2次上下文切换、0次CPU拷贝以及2次DMA拷贝。
此外，零拷贝的实现方式的技术包括基于DMA的复制和通过MMU进行内存映射，具体实现还有:
